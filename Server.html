request_game(req, res) → {Promise}

Handle /game/:gameKey request for a dump of the game. This is designed for use when opening the game interface. The game is frozen using Fridge before sending to fully encode the entire Game object, including the Players, Turn history, and the Board so they can be recreated client-side. Subsequent commands and notifications maintain the client-side game object in synch with the server game object.
