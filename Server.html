Class: Server

Server(config)

Web server for crossword game. Errors will result in an appropriate status code:

  • 404 - usually a file read error
  • 500 internal server error e.g. an assert

Routes supported:

  • GET / - Get the HTML for the game management interface
  • GET /defaults
  • GET /dictionaries
  • GET /editions
  • GET /game/:gameKey
  • GET /games/:send
  • GET /history
  • GET /locales
  • GET /theme/:css
  • GET /themes
  • POST /addRobot/:gameKey
  • POST /anotherGame/:gameKey
  • POST /createGame
  • POST /deleteGame/:gameKey
  • POST /invitePlayers/:gameKey
  • POST /join/:gameKey
  • POST /leave/:gameKey
  • POST /removeRobot/:gameKey
  • POST /sendReminder/:gameKey

See also UserManager for other user management routes.

GET_game(req, res) → {Promise}

This is designed for use when opening the game interface. The game is frozen using Fridge before sending to fully encode the entire Game object, including the Players, Turn history, and the Board so they can be recreated client-side. Subsequent commands and notifications maintain the client-side game object incrementally to keep them in synch with the server Game object.

GET_games(req, res) → {Promise}

Get a simplified version of games or a single game (no board, bag etc) for the "games" page. You can request "active" games (those still in play), "all" games (for finished games too), or a single game key. c.f. /game/:gameKey, which is used to get a full Game.
